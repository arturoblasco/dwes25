## Resultados de Aprendizaje y Criterios de Evaluación

| Resultado de Aprendizaje (RA) | Criterios de Evaluación (CE) |
|-------------------------------|------------------------------|
| **RA1. Selecciona las arquitecturas y tecnologías de programación Web en entorno servidor** | a) Diferencia modelos de ejecución cliente/servidor <br> b) Reconoce ventajas de la generación dinámica <br> c) Identifica mecanismos de ejecución en servidores Web <br> d) Reconoce funcionalidades de servidores de aplicaciones <br> e) Identifica lenguajes y tecnologías servidor <br> f) Verifica integración de lenguajes de marcas y servidor <br> g) Evalúa herramientas de programación en entorno servidor |
| **RA2. Escribe sentencias ejecutables por un servidor Web** | a) Reconoce mecanismos de generación de páginas con código embebido <br> b) Identifica tecnologías asociadas <br> c) Utiliza etiquetas para inclusión de código <br> d) Reconoce sintaxis del lenguaje de programación <br> e) Escribe y comprueba sentencias simples <br> f) Utiliza directivas para modificar comportamiento <br> g) Usa variables y operadores <br> h) Identifica ámbitos de utilización de variables |
| **RA3. Escribe bloques de sentencias embebidos en lenguajes de marcas** | a) Usa mecanismos de decisión <br> b) Emplea bucles <br> c) Utiliza arrays <br> d) Crea y usa funciones <br> e) Usa formularios web <br> f) Recupera información de formularios <br> g) Añade comentarios al código |
| **RA4. Desarrolla aplicaciones Web embebidas en lenguajes de marcas** | a) Identifica mecanismos de mantenimiento de información de cliente <br> b) Usa sesiones <br> c) Usa cookies <br> d) Identifica mecanismos de autentificación <br> e) Integra mecanismos de autentificación <br> f) Realiza adaptaciones a aplicaciones Web existentes <br> g) Usa herramientas para programación, prueba y depuración |
| **RA5. Desarrolla aplicaciones Web separando presentación y lógica de negocio** | a) Identifica ventajas de separar lógica de presentación <br> b) Analiza tecnologías y mecanismos de separación <br> c) Usa objetos y controles en servidor para generar interfaz <br> d) Usa formularios dinámicos <br> e) Configura parámetros de aplicación Web <br> f) Escribe aplicaciones con mantenimiento de estado y separación de lógica <br> g) Aplica principios de POO <br> h) Prueba y documenta código |
| **RA6. Desarrolla aplicaciones de acceso a almacenes de datos** | a) Analiza tecnologías de acceso a datos <br> b) Crea conexiones con bases de datos <br> c) Recupera información <br> d) Publica información en aplicaciones Web <br> e) Usa conjuntos de datos <br> f) Actualiza y elimina información <br> g) Usa transacciones <br> h) Prueba y documenta aplicaciones |
| **RA7. Desarrolla servicios Web** | a) Reconoce características y ámbito de los servicios Web <br> b) Reconoce ventajas de los servicios Web <br> c) Identifica tecnologías y protocolos implicados <br> d) Programa un servicio Web <br> e) Crea el documento de descripción <br> f) Verifica funcionamiento <br> g) Consume servicios Web |
| **RA8. Genera páginas Web dinámicas con tecnologías del servidor** | a) Identifica diferencias ejecución cliente/servidor <br> b) Reconoce ventajas de unir ambas tecnologías <br> c) Identifica librerías y tecnologías relacionadas <br> d) Genera páginas con interacción (alertas, confirmaciones) <br> e) Verifica formularios <br> f) Modifica dinámicamente contenido y estructura <br> g) Aplica estas tecnologías a aplicaciones Web |
| **RA9. Desarrolla aplicaciones Web híbridas** | a) Reconoce ventajas de reutilizar código e información existente <br> b) Identifica librerías y tecnologías aplicables <br> c) Crea aplicaciones que recuperan y procesan repositorios existentes <br> d) Crea repositorios específicos a partir de información externa <br> e) Usa librerías de código para funcionalidades específicas <br> f) Programa servicios/aplicaciones basados en código de terceros <br> g) Prueba, depura y documenta aplicaciones |
